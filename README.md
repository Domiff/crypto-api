# Crypto Price Tracker

Проект представляет собой клиент для криптобиржи **Deribit** и API на **FastAPI** для обработки цен на криптовалюты. 
Сервис позволяет сохранять и получать данные по тикерам `BTC_USD` и `ETH_USD` с интервалом в одну минуту.

---

## Функциональность

1. Клиент Deribit:
   - Каждую минуту запрашивает текущую **индексную цену** (`index price`) для `BTC_USD` и `ETH_USD`.
   - Сохраняет данные в базу PostgreSQL: тикер, цена и UNIX timestamp.

2. Внешнее API на FastAPI:
   - Получение всех сохраненных данных по валюте.
   - Получение последней цены валюты.
   - Получение данных по валюте с фильтром по дате.
   - Все методы используют обязательный query-параметр `ticker`.

---

## Технологии

- Python 3.14
- FastAPI
- PostgreSQL
- SQLAlchemy (async)
- Aiohttp для запросов к Deribit
- Docker и Docker Compose
- Alembic для миграций базы данных
- Uv как менеджер зависимостей
- Celery для периодического получения цен

## Установка и запуск

### 1. Подготовить директорию
```commandline
mkdir crypto-price-tracker
cd crypto-price-tracker
```

### 2. Клонировать репозиторий
```commandline
git clone https://gitlab.com/Domiff/crypto-api
```

### 3. Запуск приложения через docker compose
```commandline
docker compose up --build
```

### 4. Применить миграции для базы данных
```commandline
docker compose exec web alembic upgrade head
```

## Design decisions

1. **Асинхронность**
   - Использован `asyncio` и `asyncpg` для минимизации блокировок при запросах к Deribit и работе с PostgreSQL.
   - FastAPI работает асинхронно, что позволяет обрабатывать большое количество запросов.

2. **PostgreSQL**
   - Выбор PostgreSQL обоснован надежностью и возможностью масштабирования.
3. **Запросы к Deribit**
   - Использован `Aiohttp` с асинхронным клиентом.
   - Интервал 1 минута — баланс между актуальностью данных и нагрузкой на API.

4. **API**
   - GET-запросы для всех операций — подходят для чтения данных.
   - Query-параметр `ticker` обязательный для унификации запросов.
   - Дополнительно поддержан фильтр по времени для аналитики.

5. **Контейнеризация**
   - Обеспечивает простую установку и запуск проекта.
   - Все зависимости и база данных в контейнерах.

6. **Миграции через Alembic**
   - Позволяют безопасно управлять схемой базы данных.
